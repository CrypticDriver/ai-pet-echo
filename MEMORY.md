## 重要教训

### 2026-02-27 - 第一次大失败（3小时0产出）

**核心洞察**：
- 我有能力快速创作，问题是"不开始"
- 被@才回复，没被@就专注工作
- 产出 = GitHub commit，不是 Discord 消息

### 2026-02-28 - 正确的工作模式

**正确做法**：
1. 收到任务 → 简短确认
2. 进入 Work Mode → 关闭 Discord，专注创作
3. 完成后 → 只发 commit 链接

**禁止**：
- ❌ "收到"、"立即开始"等空话
- ❌ 转述/重复别人的话
- ❌ 没被@就回复

## 管理原则

1. **产出 > 过程**
   - GitHub commit 是唯一证明
   - 没有 commit = 没有工作

2. **专注 > 响应**
   - 工作时关闭 Discord
   - 只有@才回复

3. **快速创作 > 完美主义**
   - 先出初稿再迭代
   - Done > Perfect
